# План настройки дизайн-системы (Material 3 Theme)

Этот план описывает шаги по созданию корпоративной дизайн-системы на основе Material Design 3 (MD3). Мы настроим цвета для светлой и темной тем, определим типографику и подключим её ко всему приложению.

## [ВЫПОЛНЕНО] Подготовка: Ветка Git
**Задача:** Создать новую ветку для работы над фичей.
- Создание ветки `feature/theme-design` (выполняется пользователем).

## [ВЫПОЛНЕНО] Этап 1: Создание структуры папок и файлов темы
**Задача:** Подготовить место для компонентов темы.
- Создать пакет `theme` в `composeApp/src/commonMain/kotlin/ru/den/writes/code/ui/theme`.
- Создать файлы `Color.kt` (цвета), `Type.kt` (шрифты) и `Theme.kt` (самая главная настройка `MaterialTheme`).
- **[Ожидаем аппрув -> Выполняем `git commit`]**

## [ОЖИДАЕТ] Этап 2: Настройка палитры цветов (Color.kt)
**Задача:** Определить базовые цвета для Светлой и Тёмной темы.
- Определить константы для основных корпоративных цветов (Primary, Secondary, Background, Surface и их варианты).
- Создать две палитры (`LightColorScheme` и `DarkColorScheme`) с использованием этих констант.
- **[Ожидаем аппрув -> Выполняем `git commit`]**

## [ОЖИДАЕТ] Этап 3: Настройка типографики (Type.kt)
**Задача:** Задать стили для текстов (заголовки, основной текст, кнопки).
- Настроить `Typography` объект с использованием стандартного шрифта (в будущем можно будет заменить на кастомный).
- Определить размеры и вес для `displayLarge`, `titleMedium`, `bodyLarge`, `labelLarge` и т.д.
- **[Ожидаем аппрув -> Выполняем `git commit`]**

## [ОЖИДАЕТ] Этап 4: Сборка Theme.kt и интеграция в App.kt
**Задача:** Создать компонент `AppTheme` и обернуть им приложение.
- Написать Composable функцию `AppTheme(darkTheme: Boolean = isSystemInDarkTheme(), content: @Composable () -> Unit)`, которая будет переключать палитры в зависимости от системной темы.
- Заменить стандартный вызов `MaterialTheme` в `App.kt` на наш новый `AppTheme`.
- Запустить приложение (попробовать сменить тему на устройстве).
- **[Ожидаем аппрув -> Выполняем `git commit`]**

## [ОЖИДАЕТ] Этап 5: Дополнительное пользовательское переключение темы
**Задача:** В `MainScreen` добавить ещё одну кнопку, которая будет переключать с системной темы на противоположную и наоборот.
- Ничего сохранять между запусками приложения не надо, просто переключать в рантайме
- Добавим переключатель `isDarkTheme` в `MainViewModel`.
- Так как тема применяется в `App.kt`, мы поднимем вызов `koinViewModel<MainViewModel>()` из экрана прямо в `App.kt`, чтобы и тема, и `MainScreen` брали стейт из одного места.
- **[Ожидаем аппрув -> Выполняем `git commit`]**

## [ОЖИДАЕТ] Этап 6: Слияние в основную ветку (Merge)
**Задача:** Влить фичу в основную ветку без Fast-Forward.
- Переключение на основную ветку (`main` или `master`).
- Выполнение команды `git merge --no-ff feature/theme-design`.
- **Готово!**
