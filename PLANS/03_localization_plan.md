# План внедрения локализации (Compose Resources)

Этот план описывает шаги по переходу от захардкоженных строк к системе локализации с использованием `composeResources`. Мы вынесем все тексты из UI в XML-файлы и добавим поддержку двух языков (Английский — по умолчанию, Русский — `ru`).

## [ВЫПОЛНЕНО] Подготовка: Ветка Git
**Задача:** Создать новую ветку для работы над фичей.
- Создание ветки `feature/localization` (выполняется пользователем).

## [ВЫПОЛНЕНО] Этап 1: Создание файлов ресурсов (XML)
**Задача:** Подготовить структуру папок для локализации.
- Создание директории `composeApp/src/commonMain/composeResources/values/` для дефолтного (английского) языка и файла `strings.xml`.
- Создание директории `composeApp/src/commonMain/composeResources/values-ru/` для русского языка и файла `strings.xml`.
- **Ожидаем аппрув -> Выполняем `git commit`**

## [ВЫПОЛНЕНО] Этап 2: Наполнение ресурсов строками
**Задача:** Собрать все тексты из проекта и записать их в XML.
- Выделение строк: статусы сервера ("Подключение к серверу...", "Сервер доступен (Online)" и т.д.), текст кнопки ("Click me!").
- Добавление ключей и переводов в оба файла `strings.xml`.
- Запуск Gradle-таски для генерации KMP-кода ресурсов (`Res.string...`).
- **Ожидаем аппрув -> Выполняем `git commit`**

## [ВЫПОЛНЕНО] Этап 3: Рефакторинг UI (Использование stringResource)
**Задача:** Заменить хардкод в компонентах на вызовы KMP ресурсов.
- Обновление `ServerStatusIndicator.kt`: замена хардкода на `stringResource(Res.string.server_status_...)`.
- Обновление `MainScreen.kt`: замена текста кнопки на `stringResource(Res.string.btn_click_me)` и т.д.
- Проверка работоспособности.
- **Ожидаем аппрув -> Выполняем `git commit`**

## [ВЫПОЛНЕНО] Этап 4: Слияние в основную ветку (Merge)
**Задача:** Влить фичу в основную ветку без Fast-Forward.
- Переключение на основную ветку (`main` или `master`).
- Выполнение команды `git merge --no-ff feature/localization`.
- **Готово!**
