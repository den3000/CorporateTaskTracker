# Инструкция: Подключение мобильных приложений к локальному серверу

В этом проекте настроена автоматическая система мониторинга сети (Health Check), которая позволяет 
мобильным приложениям (Android и iOS) проверять доступность локального сервера и отображать статус в UI.

По умолчанию приложения настроены для работы **на эмуляторах/симуляторах** и автоматически используют 
правильные адреса `localhost` для каждой платформы.

Однако, если вы хотите протестировать приложение **на реальных физических устройствах (iPhone или 
Android-смартфоне)**, вам потребуется указать IP-адрес вашего компьютера (Mac) в локальной Wi-Fi сети.

Ниже описано, как работает эта система и как правильно её настраивать.

---

## 1. Как это работает под капотом

1. В корне проекта находится файл `local.properties` (он по умолчанию исключен из Git).
2. При сборке проекта (с помощью Gradle) таска `generateAppConfig` считывает параметр `SERVER_IP` 
из `local.properties` или аргументов командной строки.
3. На основе этого параметра генерируется Kotlin-объект `AppConfig.kt` (путь: 
`composeApp/build/generated/appconfig/kotlin/ru/den/writes/code/config/AppConfig.kt`).
4. Класс `NetworkMonitor` использует сгенерированную константу `AppConfig.SERVER_IP`:
   - Если параметр не задан, пуст или равен `"192.168.X.X"`, монитор автоматически переключается 
   в **режим эмулятора**:
     - Android: стучится по адресу `http://10.0.2.2:8080`.
     - iOS: стучится по адресу `http://127.0.0.1:8080`.
   - Если указан конкретный IP (например, `"192.168.0.11"`), монитор использует **режим реального 
   устройства** и стучится по адресу `http://192.168.0.11:8080`.

---

## 2. Запуск на Эмуляторах / Симуляторах (По умолчанию)

Вам **не нужно ничего настраивать**.
Оставьте параметр `SERVER_IP` пустым или равным `192.168.X.X`.

1. Запустите серверный модуль (`:server`).
2. Запустите приложение на Android эмуляторе или iOS симуляторе из Android Studio.
3. Приложение автоматически найдет сервер по локальному адресу и покажет зеленую плашку "Сервер 
доступен (Online)".

---

## 3. Запуск на Реальных Устройствах (по Wi-Fi)

Чтобы реальный телефон смог достучаться до сервера на вашем Mac, оба устройства должны быть в **одной 
Wi-Fi сети**, и приложению нужно знать IP-адрес вашего Mac.

### Шаг 3.1. Как узнать локальный IP-адрес Mac:
* **Самый быстрый способ:** Зажмите на клавиатуре клавишу `Option (Alt) ⌥` и кликните левой кнопкой 
мыши по **иконке Wi-Fi** в самой верхней строке меню Mac. Вы увидите строку **IP-адрес** (например, `192.168.0.11`).
* **Через терминал:** Введите команду 
`ipconfig getifaddr en0` или `ifconfig | grep "inet " | grep -v 127.0.0.1`.

### Шаг 3.2. Как передать IP-адрес приложению:

У вас есть два способа это сделать.

#### Способ А: Через файл `local.properties` (Рекомендуемый для постоянной работы)
1. Откройте файл `local.properties` в корне проекта (если его нет — создайте).
2. Добавьте или измените строку:
   ```properties
   SERVER_IP=192.168.0.11
   ```
3. Соберите и запустите приложение на физическом устройстве прямо из Android Studio.
   *Приложение подхватит ваш IP, и вы увидите, что оно успешно подключилось к серверу.*

#### Способ Б: Через терминал (Для разовой проверки или CI/CD)
Если вы не хотите менять файлы, вы можете передать IP-адрес напрямую как аргумент Gradle (`Project Property`):
```bash
# Для сборки и установки Debug-версии Android-приложения:
./gradlew installDebug -PSERVER_IP=192.168.0.11
```
*(Этот параметр переопределит значение из `local.properties`, если оно там есть).*

---

## 4. Разрешения (Уже настроены)
Для работы локальной сети по HTTP мы добавили следующие разрешения (они уже есть в проекте):
* **Android:** В `AndroidManifest.xml` включен флаг `android:usesCleartextTraffic="true"`, 
разрешающий HTTP запросы.
* **iOS:** В `Info.plist` добавлено разрешение `NSAppTransportSecurity -> NSAllowsArbitraryLoads`, 
а также `NSLocalNetworkUsageDescription`.
* *Внимание: При первом запуске на реальном iPhone появится системное диалоговое окно с просьбой 
разрешить приложению доступ к устройствам в локальной сети. Обязательно нажмите **"Разрешить"**, 
иначе соединение будет заблокировано.*
